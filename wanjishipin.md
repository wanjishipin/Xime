# 玩机视频 二改功能说明

本文件记录了基于原始 Xime 输入法（作者：kingzcheung）进行的二次开发新增功能和修复。

原始项目：https://github.com/kingzcheung/Xime
二改源代码：https://github.com/wanjishipin/Xime
玩机视频主页：https://b23.tv/wfoztmK

---

## 新增功能

### 1. 听录功能
在定制工具栏新增听录按钮，实现实时字幕和语音转文字记录。
- 持续语音识别，累积文本，支持实时字幕展示
- 支持本地 ASR 引擎和在线引擎（阿里百炼）
- 本地引擎支持标点推理模型，自动为识别结果添加标点
- 停止后统一对全文推理标点，避免标点滞后
- 支持复制识别结果、清空记录

### 2. 剪贴板导入导出
在剪贴板设置页面新增导入导出按钮。
- 导出：将剪贴板历史记录导出为 JSON 文件
- 导入：从 JSON 文件恢复剪贴板记录

### 3. WebDAV 全量备份
在 WebDAV 同步页面新增全量备份开关。
- 开启后，上传/下载操作将打包应用全部私有数据（filesDir）为 zip 进行备份/恢复
- 包含 rime 方案、模型配置、用户词典、剪贴板等所有私有数据
- 关闭时保持原有 rime 方案同步行为

### 4. 特殊按键支持
键盘新增 Ctrl、Alt、Tab 等控制按键，以及上下方向键快速输入。

### 5. 剪贴板搜索增强
- 搜索模式默认显示全部剪贴板列表
- 修复搜索框退格、中文输入等问题
- 缩小搜索结果列表高度，避免压缩键盘

### 6. 置顶按钮
实现置顶按钮功能，即使无输入焦点也能保持键盘显示。

### 7. 剪贴板最大条数设置
新增剪贴板最大条数设置项，可自定义保留的剪贴板记录数量。

## Bug 修复

### 1. WebDAV 不支持 HTTP 明文连接
修复 WebDAV 备份在 HTTP（非 HTTPS）环境下报错 `CLEARTEXT communication not permitted` 的问题，新增 network_security_config 允许明文流量。

### 2. 听录启动后立即停止
修复听录功能启动后「正在听」马上消失的问题。根因是 `preload()` 后台线程与 `startRecognition()` 主线程竞争 backend，改为直接调 `startRecognition()`，backend 按需创建。

### 3. 听录标点推理滞后
修复长段语音识别结果前面全部无标点、标点只在最后产生的问题。改为累积 rawSegments，每次新结果或模型就绪时对全文统一推理标点。

### 4. 听录停止后内容重复
修复点击停止后补上标点时内容重复的问题。根因是 `stop()` 同步添加 partialText 为 segment，异步 `handleResult()` 又添加 final result 为 segment，两者为同一 session 的累积文本。通过 `isStopping` 标志位避免重复添加。

### 5. Ctrl/Alt 按钮高度异常
修复 Ctrl/Alt 按钮高度变矮的问题，添加 `fillMaxHeight`。

## 其他改动

### 1. 包名变更
应用包名改为 `com.wanjishipin.xime`，插件包名同步修改，目录结构不变。

### 2. 关于页面
- 原作者改为「原始作者」，保留原始作者源代码链接
- 新增「二改作者」信息栏，包含玩机视频信息和二改源代码链接

### 3. 应用名称
安装包应用名称添加「玩机视频@bilibili」标识。

### 4. Debug 签名
Debug 版本支持使用 `app/keystore.properties` 签名，使用 UTF-8 编码读取配置以支持中文路径和别名。
